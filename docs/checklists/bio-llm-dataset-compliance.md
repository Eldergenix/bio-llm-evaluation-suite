# Bio-LLM Dataset Compliance Checklist

Use this checklist before incorporating any dataset into the Bio-LLM Evaluation Suite. Update the table with dataset-specific details and file completed checklists under `docs/checklists/completed/`.

| Item | Description | Dataset Owner | Status | Notes |
| --- | --- | --- | --- | --- |
| Licensing | Dataset license permits research & derivative works? |  | ☐ | Include citation requirements. |
| Redistribution | Redistribution allowed (or alternative access instructions provided)? |  | ☐ | Document download/approval process. |
| PHI/PII | Dataset free of PHI/PII or de-identified per HIPAA Safe Harbor? |  | ☐ | Record de-identification method. |
| Data Use Agreements | Required DUAs signed/stored? |  | ☐ | Link to executed DUA or request log. |
| Security Controls | Storage location compliant with org policy (encryption at rest, access controls)? |  | ☐ | Note bucket path / ACLs. |
| Provenance | Source verified and checksums recorded? |  | ☐ | Attach checksum manifest. |
| Bias Review | Known demographic/clinical biases documented? |  | ☐ | Capture mitigation strategies. |
| Ethical Review | IRB or internal ethics approval required? |  | ☐ | Provide approval ID or justification. |
| Regulatory Alignment | Dataset meets HIPAA, GDPR, or regional requirements? |  | ☐ | Note jurisdictional considerations. |
| Update Cadence | Document how updates/patches are tracked. |  | ☐ | Link to upstream changelog. |
| Usage Logging | Access + usage logging configured? |  | ☐ | Include log retention policy. |
| Data Retention | Define retention + deletion policy (timeline, triggers). |  | ☐ | Specify automation steps. |
| Documentation | Readme/metadata available (schema, units, preprocessing). |  | ☐ | Link to documentation. |
| Consent | Confirm data aligns with participant consent statements. |  | ☐ | Reference consent language. |
| External Sharing | Outline rules for sharing derived artefacts (models, embeddings). |  | ☐ | Note restrictions. |
